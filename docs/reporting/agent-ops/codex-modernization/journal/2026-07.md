## 2026-07-01 Dashboard Tracker Compatibility Fallback Telemetry

- Added structured client-side dashboard telemetry for legacy tracker fallback
  from typed APIs. The dashboard now emits
  `pixeagle:tracker-compatibility-fallback`, records a bounded in-memory
  history of the last 50 attempted fallback events, and logs a structured
  warning when:
  - typed tracker catalog fallback serves selector/current-config metadata from
    legacy `/api/tracker/available-types` and `/api/tracker/current-config`;
  - typed available-trackers catalog fallback uses legacy
    `/api/tracker/available`;
  - typed current-tracker catalog fallback uses legacy
    `/api/tracker/current`;

- Kept fallback limited to missing or explicitly unsupported typed routes
  (`404`, `405`, `501`, or existing typed unavailable/no-entry handling).
  Auth, policy, and malformed typed payload failures still do not fall back.

  state that deprecated `/api/tracker/set-type` is compatibility-only and that
  new tracker-selection clients should use
