## 2026-07-01 Typed Tracker Switch Action

- Added typed `POST /api/v1/actions/tracker-switch` for tracker selection.
  The action uses the existing action-resource contract: dry-run validation,
  explicit `confirm=true`, required idempotency keys for confirmed mutations,
  process-local action records, structured typed errors, and security-critical
  action-execute policy coverage.
- Extracted `switch_tracker_to_type()` from the legacy tracker route helper so
  the typed action can reuse current schema-manager validation and
  `AppController.switch_tracker_type()` behavior without fabricating a FastAPI
  request. Legacy `/api/tracker/switch` response shapes are preserved for
  rolling-update compatibility.
- Dashboard tracker selection now uses the typed action through the endpoint
  registry and existing action request envelope. It falls back to legacy
  `/api/tracker/switch` only when the typed action is missing or explicitly
  unsupported, and it no longer uses deprecated `/api/tracker/set-type`.
- Independent read-only review found and closed one dashboard blocker: typed
  action failures with object-shaped `detail` payloads could be rendered as raw
  React children. The hook now normalizes nested typed error envelopes to
  strings, and the policy-failure regression uses an object-shaped typed
  `detail`.
- Generated API/MCP candidate inventory now includes
  `pixeagle.actions.tracker_switch.submit_candidate` as blocked, unregistered,
  non-callable, and excluded from docs-stage `agent_tools.yaml`.
